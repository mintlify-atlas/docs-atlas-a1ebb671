Better Todo uses [Resend](https://resend.com) to send transactional emails for board invitations, card assignments, and mentions.

## Resend Setup

### Getting an API Key

<Steps>
  <Step title="Create a Resend account">
    Sign up at [resend.com](https://resend.com) if you don't have an account.
  </Step>
  
  <Step title="Generate an API key">
    Go to Settings → API Keys → Create API Key. Copy the key (it starts with `re_`).
  </Step>
  
  <Step title="Configure environment variable">
    Set the API key in Convex:
    
    ```bash
    npx convex env set RESEND_API_KEY "re_your_api_key_here"
    ```
  </Step>
</Steps>

### FROM_EMAIL Configuration

Configure the sender email address for all outgoing emails:

```bash
npx convex env set FROM_EMAIL "noreply@yourdomain.com"
```

<Warning>
For production use, verify your domain in Resend to send from your own domain. Otherwise, emails will be sent from `onboarding@resend.dev`.
</Warning>

### Environment Variables

<ParamField path="RESEND_API_KEY" type="string" required>
  Your Resend API key. Starts with `re_`. Required for sending emails.
</ParamField>

<ParamField path="FROM_EMAIL" type="string" default="onboarding@resend.dev">
  The email address that appears in the "From" field. Use a verified domain in production.
</ParamField>

## Board Invite Emails

Two types of board invite emails are sent depending on whether the recipient is already registered.

### For Registered Users

Sent via `sendBoardInviteEmail` when inviting existing users.

From `emails.ts:114-184`:

```typescript
export const sendBoardInviteEmail = internalAction({
    args: {
        to: v.string(),
        recipientName: v.optional(v.string()),
        inviterName: v.string(),
        boardTitle: v.string(),
        boardId: v.string(),
        role: v.string(),
        inviteId: v.string(),
    },
    handler: async (_ctx, args) => {
        const resend = getResend();
        const acceptUrl = `${siteUrl}/boards/${args.boardId}?acceptInvite=${args.inviteId}`;
        
        await resend.emails.send({
            from: getFromEmail(),
            to: args.to,
            subject: `${args.inviterName} invited you to "${args.boardTitle}" on BetterTodo`,
            html: baseTemplate(content, previewText),
        });
    },
});
```

**Email includes**:
- Personalized greeting with recipient's name
- Inviter's name and board title
- Role badge (admin/member/viewer)
- Direct link to accept invitation
- Board information card

Sent via `sendBoardInviteEmailExternal` for users who don't have an account yet.

<Steps>
  <Step title="User receives email">

- Direct link to view the card
- Context card with board/card information

## Production Configuration

### Domain Verification

<Steps>
  <Step title="Add domain in Resend">
    Go to Resend → Domains → Add Domain and enter your domain name.
  </Step>
  
  <Step title="Add DNS records">
    Add the provided DNS records (SPF, DKIM, DMARC) to your domain's DNS settings.
  </Step>
  
  <Step title="Verify domain">
    Wait for verification (usually 5-10 minutes) and click Verify in Resend.
  </Step>
  
  <Step title="Update FROM_EMAIL">
    Set your verified domain email:
    
    ```bash
    npx convex env set FROM_EMAIL "noreply@yourdomain.com" --prod
    ```
  </Step>
</Steps>

### Best Practices

<Warning>
**Important for deliverability**:
- Use a verified domain in production
- Set up SPF, DKIM, and DMARC records
- Monitor bounce rates in Resend dashboard
- Avoid sending too many emails too quickly
</Warning>

## Troubleshooting

### Emails Not Sending

1. **Check API key**: Verify `RESEND_API_KEY` is set correctly
   ```bash
   npx convex env get RESEND_API_KEY
   ```

2. **Check Resend logs**: Visit dashboard to see error messages

3. **Verify environment**: Ensure `SITE_URL` is set for generating links

### Emails Going to Spam

1. **Verify your domain** with proper DNS records
2. **Warm up your domain** by starting with low volume
3. **Check content** for spam trigger words
4. **Test with Mail Tester** at [mail-tester.com](https://www.mail-tester.com)

### Template Rendering Issues

1. **Test in Litmus** or Email on Acid for client compatibility
2. **Use inline CSS** (already done in templates)
3. **Avoid external images** or use absolute URLs
4. **Check mobile rendering** in Gmail app and iOS Mail
