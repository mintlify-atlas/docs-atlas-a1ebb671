# Welcome to Better Todo

Better Todo is a powerful Trello-like collaborative task management application that helps teams organize, track, and complete their work efficiently. Built with modern web technologies, it provides a seamless real-time collaboration experience with a beautiful, intuitive interface.

## What is Better Todo?

Better Todo combines the simplicity of Kanban boards with powerful features like real-time collaboration, notifications, and rich card details. Whether you're managing a small project or coordinating a large team, Better Todo adapts to your workflow.

## Tech Stack

Better Todo is built with cutting-edge technologies that ensure performance, scalability, and developer experience:

### Frontend

- **React 19** - The latest version of React with improved performance
- **TanStack Router** - Type-safe routing with nested layouts
- **TailwindCSS** - Utility-first CSS framework
- **shadcn/ui** - Beautiful, accessible component library
- **@hello-pangea/dnd** - Smooth drag-and-drop interactions

### Backend

- **Convex** - Backend-as-a-Service with real-time reactivity
- **Better Auth** - Modern authentication with email and OAuth support
- **Resend** - Transactional email service

### Infrastructure

- **Turborepo** - High-performance monorepo build system
- **Bun** - Fast JavaScript runtime and package manager
- **Vite** - Lightning-fast frontend build tool

<Tip>
  Better Todo uses Convex for the backend, which means you get real-time updates, serverless functions, and a type-safe database all in one platform.
</Tip>

## Architecture Overview

Better Todo follows a monorepo architecture with clear separation of concerns:

```
BetterTodo/
├── apps/
│   └── web/              # React frontend application
│       ├── components/   # UI components (Board, Card, List)
│       ├── routes/       # TanStack Router routes
│       └── lib/          # Auth client, utilities, constants
├── packages/
│   ├── backend/          # Convex backend
│   │   └── convex/       # Functions, schema, auth config
│   ├── config/           # Shared TypeScript configuration
│   └── env/              # Environment schema validation
```

## Core Concepts

<AccordionGroup>
  <Accordion title="Boards">
    Boards are the top-level organizational unit. Each board contains lists and cards, and can be private, team-based, or public. Boards support custom colors, backgrounds, and member permissions.
  </Accordion>
  
  <Accordion title="Lists">
    Lists organize cards within a board. They represent stages in your workflow (e.g., "To Do", "In Progress", "Done"). Lists can be reordered and have optional card limits.
  </Accordion>
  
  <Accordion title="Cards">
    Cards represent individual tasks or items. They support rich descriptions, due dates, priorities, labels, checklists, comments, attachments, and member assignments.
  </Accordion>
  
  <Accordion title="Workspaces">
    Workspaces group related boards together. They help teams organize multiple projects and control access at a higher level.
  </Accordion>
</AccordionGroup>
