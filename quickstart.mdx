# Quickstart Guide

This guide will walk you through setting up Better Todo on your local machine. You'll have a fully functional task management application running in just a few minutes.

<Note>
  This quickstart assumes you have Bun or Node.js 18+ installed. If you don't, check out the [installation guide](/installation) for prerequisites.
</Note>

## Getting Started

<Steps>
  <Step title="Clone the repository">
    First, clone the Better Todo repository to your local machine:
    
    ```bash
    git clone https://github.com/yourusername/BetterTodo.git
    cd BetterTodo
    ```
  </Step>

  <Step title="Install dependencies">
    Install all project dependencies using Bun (or npm/yarn if you prefer):
    
    <CodeGroup>
    ```bash Bun
    bun install
    ```
    
    ```bash npm
    npm install
    ```
    
    ```bash yarn
    yarn install
    ```
    </CodeGroup>
    
    <Tip>
      Bun is significantly faster than npm or yarn. We recommend using it for the best development experience.
    </Tip>
  </Step>

  <Step title="Set up Convex">
    Run the automated Convex setup script. This will create a new Convex project and connect it to your local development environment:
    
    ```bash
    bun run dev:setup
    ```
    
    This command will:
    - Create a new Convex project (or connect to an existing one)
    - Generate a `.env.local` file in `packages/backend/`
    - Set up the Convex development server
    
    <Note>
      You'll need to log in to Convex if this is your first time. The CLI will guide you through the authentication process.
    </Note>
  </Step>

  <Step title="Configure environment variables">
    Copy the Convex environment variables from `packages/backend/.env.local` to `apps/web/.env` if needed. Then set the following environment variables in your Convex project:
    
    ```bash
    npx convex env set SITE_URL "http://localhost:3001"
    npx convex env set GOOGLE_CLIENT_ID "your-google-client-id"
    npx convex env set GOOGLE_CLIENT_SECRET "your-google-client-secret"
    npx convex env set RESEND_API_KEY "re_your_resend_api_key"
    npx convex env set FROM_EMAIL "noreply@yourdomain.com"
    ```
    
    <Warning>
      **Google OAuth Setup**: To enable Google authentication, you need to create OAuth credentials in the [Google Cloud Console](https://console.cloud.google.com/apis/credentials). Add the redirect URI: `https://<your-deployment>.convex.site/api/auth/callback/google`
    </Warning>
    
    <Accordion title="Optional: Email notifications">
      To enable email notifications for board invites and @mentions, you'll need a [Resend](https://resend.com) API key. You can skip this for local development - the app will work without it.
    </Accordion>
  </Step>

  <Step title="Start the development server">
    Start all development servers (frontend and Convex backend):
    
    ```bash
    bun run dev
    ```
    
    This will start:
    - **Web app** at [http://localhost:3001](http://localhost:3001)
    - **Convex backend** with hot reloading
    
    <Tip>
      You can also start individual services:
      - `bun run dev:web` - Frontend only
      - `bun run dev:server` - Convex backend only
    </Tip>
  </Step>

  <Step title="Create your first admin user">
    Open [http://localhost:3001](http://localhost:3001) in your browser and sign up with your email. After creating your account:
    
    1. Go to the [Convex Dashboard](https://dashboard.convex.dev)
    2. Navigate to your project
    3. Click on **Functions**
    4. Find and run the `auth:createAdmin` internal mutation
    5. Pass your `userId` to grant yourself admin privileges
    
    <Note>
      You can find your `userId` by checking the browser console after logging in, or by querying the users table in the Convex dashboard.
    </Note>
  </Step>
</Steps>

## Available Scripts

Here are the key commands you'll use during development:

| Command | Description |
| --- | --- |
| `bun run dev` | Start all apps (web + Convex) |
| `bun run build` | Build all applications for production |
| `bun run dev:web` | Start only the web app |
| `bun run dev:server` | Start only the Convex backend |
| `bun run dev:setup` | Setup and configure Convex project |
| `bun run check-types` | Type-check all packages |

## Troubleshooting

<AccordionGroup>
  <Accordion title="Convex connection errors">
    If you're having trouble connecting to Convex:
    
    1. Make sure you're logged in: `npx convex dev --login`
    2. Check that your `.env.local` file exists in `packages/backend/`
    3. Verify the `CONVEX_DEPLOYMENT` variable is set correctly
    4. Try restarting the dev server
  </Accordion>
  
  <Accordion title="Port already in use">
    If port 3001 is already in use:
    
    1. Kill the process using the port: `lsof -ti:3001 | xargs kill -9`
    2. Or change the port in `apps/web/vite.config.ts`
  </Accordion>
  
  <Accordion title="Module not found errors">
    If you see module resolution errors:
    
    1. Delete `node_modules` and reinstall: `rm -rf node_modules && bun install`
    2. Clear Bun cache: `bun pm cache rm`
    3. Make sure you're using Bun 1.2.21 or later: `bun --version`
  </Accordion>
  
  <Accordion title="Authentication not working">
    If you can't log in:
    
    1. Verify `SITE_URL` is set to `http://localhost:3001` in Convex env
    2. Check that Better Auth is configured correctly in `packages/backend/convex/auth.ts`
    3. For Google OAuth, ensure the redirect URI matches your Convex deployment
  </Accordion>
</AccordionGroup>
